## 📖 功能一：页面显示美化内容

### 🎯 功能说明

美化完成后，不再需要下载文件才能查看效果，直接在页面上就能看到：
- **原始内容**：纯文本显示
- **美化后**：富文本渲染（Markdown → HTML）

### 💡 使用方法

#### 方法 A: 上传文件
1. 点击"📁 上传文件"标签
2. 拖拽或点击选择 `.md` 文件
3. 选择模型（如：通义千问 Turbo）
4. 点击"✨ 开始美化"
5. 等待处理完成
6. 在结果区域点击"美化后"标签
7. 查看渲染后的美化效果

#### 方法 B: 直接输入
1. 点击"✏️ 直接输入"标签
2. 粘贴 Markdown 内容
3. 选择模型
4. 点击"开始美化"
5. 查看渲染效果

### 🎨 渲染效果

系统会自动渲染以下 Markdown 元素：

| Markdown | 渲染效果 |
|----------|----------|
| `# 标题` | `<h1>` 大标题 |
| `## 标题` | `<h2>` 中标题 |
| `**粗体**` | **粗体文字** |
| `*斜体*` | *斜体文字* |
| `` `代码` `` | `行内代码` |
| ```代码块``` | 代码块高亮 |
| `- 列表` | 无序列表 |
| `1. 有序` | 有序列表 |

### 📋 复制内容

点击"📋 复制内容"按钮，可以一键复制美化后的 Markdown 文本。

## 🔑 功能二：页面配置 API KEY

### 🎯 功能说明

无需手动编辑 `.env` 文件，在网页上就能配置所有主流 AI 平台的 API 密钥。

### 💡 使用步骤

1. **打开配置面板**
   - 点击右上角的 "⚙️ API 配置" 按钮

2. **填写 API Key**
   
   **OpenAI:**
   ```
   API Key: sk-xxxxxxxxxxxxxxxx
   ```
   
   **通义千问（推荐）:**
   ```
   API Key: sk-xxxxxxxxxxxxxxxx
   ```
   
   **文心一言:**
   ```
   Access Key: xxxxxxxx
   Secret Key: xxxxxxxx
   ```
   
   **讯飞星火:**
   ```
   API Key: xxxxxxxx
   ```
   
   **智谱 AI:**
   ```
   API Key: xxxxxxxx
   ```
   
   **腾讯混元:**
   ```
   API Key: xxxxxxxx
   ```

3. **保存配置**
   - 点击 "💾 保存配置" 按钮
   - 看到 "✅ 配置已保存！" 提示

4. **验证配置**
   - 刷新页面
   - 查看模型列表中对应模型的可用状态
   - ✅ 表示可用，❌ 表示未配置

### ⚠️ 注意事项

1. **会话级有效**
   - 配置仅在当前浏览器会话中有效
   - 刷新页面后需要重新配置
   - 关闭浏览器后配置清除

2. **永久配置**
   - 如需永久保存，请编辑 `.env` 文件
   - 参考 `.env.example` 示例

3. **安全性**
   - 配置保存在本地内存
   - 不会上传到服务器
   - 刷新后即失效

## 💬 功能三：AI 聊天窗口

### 🎯 功能说明

右下角悬浮聊天窗口，支持选择不同的大模型进行智能对话。

### 💡 使用步骤

1. **打开聊天窗口**
   - 点击右下角的 "💬 聊天" 浮动按钮
   - 聊天窗口弹出

2. **选择模型**
   - 在聊天窗口顶部下拉菜单中选择模型
   - 可选模型取决于已配置的 API Key

3. **开始对话**
   - 在输入框中输入问题
   - 按 `Enter` 发送（`Shift+Enter` 换行）
   - 等待 AI 回复

4. **切换模型**
   - 在下拉菜单中选择其他模型
   - 继续提问，对比不同模型的回答风格

## ❓ 常见问题

### Q1: 配置保存后找不到？
**A:** 配置是会话级的，刷新页面会丢失。这是正常行为。

### Q2: 聊天窗口打不开？
**A:** 
1. 检查浏览器控制台是否有错误
2. 确认 JavaScript 已启用
3. 刷新页面重试

### Q3: 美化后的内容显示不正常？
**A:** 
1. 切换到"原始内容"查看源码
2. 检查 Markdown 格式是否正确
3. 下载后用专业编辑器查看

### Q4: 聊天没反应？
**A:**
1. 检查是否选择了模型
2. 确认该模型的 API Key 已配置
3. 查看网络请求是否成功

### Q5: 可以同时使用多个功能吗？
**A:** 可以！三个功能互不干扰，可以同时使用。
