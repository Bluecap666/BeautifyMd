# 🌐 Web UI 使用指南

MD BeautifyArts v2.0 提供了友好的 Web 界面，让你可以在线美化 Markdown 文档。

---

## 🚀 快速启动

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

确保已安装 Flask 相关依赖：
- flask>=3.0.0
- flask-cors>=4.0.0
- werkzeug>=3.0.0

### 2. 配置模型

编辑 `.env` 文件，配置至少一个模型的 API 密钥：

```ini
# 选择其中一个配置即可

# OpenAI
OPENAI_API_KEY=sk-your-key

# 或通义千问
DASHSCOPE_API_KEY=your-dashscope-key

# 或文心一言
QIANFAN_AK=your-ak
QIANFAN_SK=your-sk
```

### 3. 启动 Web 服务

```bash
python web_app.py
```

看到以下输出表示成功：
```
╔══════════════════════════════════════════════════════╗
║                                                      ║
║     MD BeautifyArts - Web UI Server                  ║
║                                                      ║
║     🚀 服务已启动                                    ║
║                                                      ║
╚══════════════════════════════════════════════════════╝
    
    访问地址：http://localhost:5000
```

### 4. 访问界面

打开浏览器访问：**http://localhost:5000**

## 💻 界面功能

### 1. 模型选择区
- 🤖 显示所有可用模型
- 按提供商分组（OpenAI、通义千问、文心一言等）
- 自动检测已配置的模型
- 实时切换要使用的模型

### 2. 上传方式
提供两种美化方式：

#### 📁 上传文件模式
- 点击上传区域选择文件
- 拖拽文件到上传区域
- 支持 .md 格式
- 最大支持 16MB 文件

#### ✏️ 直接输入模式
- 在大文本框中输入/粘贴 Markdown
- 适合短内容快速美化
- 实时预览

### 3. 美化处理
- ✨ 智能添加 Emoji
- 📏 优化分割线
- 📝 格式化代码块
- 🎨 美化列表和表格
- 🔧 修复格式问题

### 4. 结果展示
- 📄 原始内容和美化后内容对比
- 💾 下载美化后的文件
- 📋 一键复制内容
- 🔄 重新开始新任务

## 📖 使用步骤

### 完整流程

#### 步骤 1: 选择模型
从下拉菜单中选择一个可用的 AI 模型。

**推荐选择：**
- 国内用户：通义千问、文心一言、讯飞星火
- 追求质量：GPT-4
- 性价比：GPT-3.5、ChatGLM Lite

#### 步骤 2: 上传内容
**方式一：上传文件**
1. 点击"📁 上传文件"标签
2. 点击上传区域或拖拽文件
3. 选择你的 Markdown 文件

**方式二：直接输入**
1. 点击"✏️ 直接输入"标签
2. 在文本框中粘贴或输入 Markdown
3. 内容会实时保存

#### 步骤 3: 开始美化
点击"✨ 开始美化"按钮，等待 AI 处理。

处理时间参考：
- 小文件 (<10KB): 5-10 秒
- 中文件 (10-50KB): 10-30 秒
- 大文件 (>50KB): 30 秒 -2 分钟

#### 步骤 4: 查看结果
美化完成后：
1. 切换到"美化后"标签查看效果
2. 对比原始内容和美化效果
3. 满意后下载或复制

## 🎯 实用技巧

### 技巧 1: 批量处理
虽然 Web UI 一次处理一个文件，但你可以：
1. 保持页面打开
2. 连续上传多个文件
3. 依次处理并下载

### 技巧 2: 预览对比
使用预览功能对比美化和前后的差异：
- 点击"原始内容"查看原文
- 点击"美化后"查看效果
- 确保美化符合预期

### 技巧 3: 快速复制
对于短文档：
1. 使用"直接输入"模式
2. 美化后点击"📋 复制内容"
3. 粘贴到你的编辑器

### 技巧 4: 模型切换
如果某个模型效果不理想：
1. 在下拉菜单切换其他模型
2. 重新上传文件美化
3. 对比不同模型的效果

## ⚙️ 高级配置

### 自定义端口

修改 `web_app.py` 中的端口：
```python
app.run(debug=True, host='0.0.0.0', port=8080)  # 改为 8080
```

### 远程访问

默认绑定 `0.0.0.0`，允许远程访问：
```bash
# 局域网内其他设备可访问
http://你的 IP:5000
```

### 生产环境部署

使用 Gunicorn 部署：
```bash
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 web_app:app
```

## 🔍 API 接口说明

Web UI 提供了 RESTful API，可以直接调用。

### 获取模型列表
```http
GET /api/models
```

响应：
```json
{
  "success": true,
  "models": [
    {
      "id": "gpt-3.5-turbo",
      "name": "GPT-3.5 Turbo",
      "provider": "OpenAI",
      "available": true
    }
  ]
}
```

### 美化文件
```http
POST /api/beautify
Content-Type: multipart/form-data

参数:
- file: Markdown 文件
- model: 模型 ID（可选）
```

### 美化文本
```http
POST /api/beautify-text
Content-Type: application/json

{
  "content": "# Markdown 内容",
  "model": "gpt-3.5-turbo"
}
```

### 下载文件
```http
GET /api/download/<filename>
```

## ❓ 常见问题

### Q1: 页面打不开？
**A:** 
1. 检查 Flask 是否安装：`pip list | grep flask`
2. 确认端口未被占用
3. 查看终端错误信息

### Q2: 模型列表为空？
**A:** 
1. 检查 `.env` 文件是否存在
2. 确认至少配置了一个模型的密钥
3. 刷新页面重试

### Q3: 上传失败？
**A:** 
1. 检查文件大小（最大 16MB）
2. 确认文件格式为 .md
3. 查看浏览器控制台错误

### Q4: 美化时间过长？
**A:** 
1. 文件可能较大，耐心等待
2. 网络可能较慢
3. 考虑升级更快的模型

### Q5: 效果不满意？
**A:** 
1. 尝试切换其他模型
2. 调整配置文件中的美化选项
3. 使用命令行模式自定义

## 🛠️ 故障排除

### 查看日志
Web 服务的日志会输出到终端，包含：
- 请求信息
- 处理状态
- 错误详情

### 常见错误

**错误 1: ModuleNotFoundError**
```bash
# 解决：安装缺失的包
pip install flask flask-cors
```

**错误 2: Address already in use**
```bash
# 解决：更换端口或关闭占用进程
# Windows:
netstat -ano | findstr :5000
taskkill /PID <进程 ID> /F

# Linux/Mac:
lsof -i :5000
kill -9 <进程 ID>
```

**错误 3: API Key Error**
```
# 解决：检查.env 配置
# 确认密钥正确且未过期
```
